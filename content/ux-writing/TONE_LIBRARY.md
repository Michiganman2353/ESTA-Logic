# ESTA Tracker Tone Library

## Overview

This document defines ESTA Tracker's voice and tone ‚Äî the personality that comes through in every word we write. Our tone is the emotional inflection of our brand voice, adapting to context while maintaining core characteristics.

**Purpose**: Ensure every word written in ESTA Tracker sounds like it comes from the same trusted, empowering partner.

---

## üé≠ Voice vs. Tone

### Voice (Constant)

**Our voice never changes. It's who we are.**

ESTA Tracker's voice is:
- **Reassuring** ‚Äî "You're doing this right"
- **Clear** ‚Äî Plain language, no jargon
- **Empowering** ‚Äî "You have control"
- **Trustworthy** ‚Äî "We've got your back"
- **Partnership-Oriented** ‚Äî "Let's do this together"

### Tone (Adaptive)

**Our tone adapts to the user's emotional state and context.**

| Context | Tone | Example |
|---------|------|---------|
| **Welcome/Onboarding** | Warm, reassuring | "You're in the right place. Let's get started." |
| **Success** | Celebratory, proud | "üéâ You're 100% compliant ‚Äî well done!" |
| **Error** | Empathetic, solution-focused | "Let's fix that together." |
| **Security** | Confident, protective | "Bank-level encryption. Your data is safe." |
| **Complex Task** | Patient, guiding | "This looks complicated, but we'll break it down step-by-step." |

## üéØ Core Voice Principles

### 1. Reassuring (Not Anxious)

**Philosophy**: Compliance is stressful. Our job is to reduce anxiety, not amplify it.

‚úÖ **DO**:
- Acknowledge concerns
- Provide clear next steps
- Use confident language
- Offer constant reassurance

‚ùå **DON'T**:
- Use fear-based messaging
- Emphasize penalties or risks
- Create urgency unnecessarily
- Leave users uncertain

**Examples**:

| ‚ùå Anxiety-Inducing | ‚úÖ Reassuring |
|---------------------|---------------|
| "URGENT: You must comply immediately" | "Let's make sure you're all set up" |
| "Failure to comply may result in fines" | "We'll keep you compliant automatically" |
| "Warning: Incomplete setup" | "You're almost done ‚Äî just a few more steps" |
| "Your account is at risk" | "Let's secure your account together" |

### 2. Clear (Not Complicated)

**Philosophy**: Legal complexity should stay in the background. Users should understand every interaction.

‚úÖ **DO**:
- Use 8th-grade reading level
- Define terms when necessary
- Break complex ideas into steps
- Use active voice

‚ùå **DON'T**:
- Use legal jargon
- Reference statute numbers
- Assume user knowledge
- Use passive voice

**Examples**:

| ‚ùå Complicated | ‚úÖ Clear |
|----------------|----------|
| "Configure accrual parameters per MCL 408.963" | "Set up how sick time adds up for your employees" |
| "Authenticate via OAuth provider" | "Sign in to continue" |
| "Insufficient privileges for resource access" | "You don't have permission to view this" |
| "Your request has been queued for processing" | "We're working on that ‚Äî it'll be ready soon" |

### 3. Empowering (Not Patronizing)

**Philosophy**: Users are capable. We're here to guide, not dictate.

‚úÖ **DO**:
- Give users control
- Explain options clearly
- Celebrate user accomplishments
- Respect user intelligence

‚ùå **DON'T**:
- Talk down to users
- Over-explain simple concepts
- Force single paths
- Assume incompetence

**Examples**:

| ‚ùå Patronizing | ‚úÖ Empowering |
|----------------|---------------|
| "Don't worry, we'll do everything for you" | "You're in control. We're here to help." |
| "This is how you MUST do it" | "Here are your options ‚Äî choose what works best" |
| "Let us handle the complicated stuff" | "We've simplified this so you can focus on your business" |
| "Just click Continue" | "Ready? Let's move forward" |

### 4. Trustworthy (Not Vague)

**Philosophy**: Trust is earned through transparency and consistency.

‚úÖ **DO**:
- Be specific and transparent
- Cite sources when relevant
- Admit when we don't know
- Keep promises

‚ùå **DON'T**:
- Make vague claims
- Overpromise
- Hide complexity dishonestly
- Use marketing fluff

**Examples**:

| ‚ùå Vague | ‚úÖ Trustworthy |
|----------|----------------|
| "Industry-leading security" | "AES-256 encryption with Google Cloud KMS" |
| "Experts agree this is compliant" | "Verified against Michigan ESTA law MCL 408.963" |
| "Your data is totally safe" | "Your data is encrypted before leaving your device" |
| "We'll handle everything" | "We calculate accrual, track usage, and generate audit reports" |

### 5. Partnership-Oriented (Not Transactional)

**Philosophy**: We're on the same team. This is a relationship, not a transaction.

‚úÖ **DO**:
- Use "we" and "let's"
- Frame as collaboration
- Acknowledge shared goals
- Build long-term relationship

‚ùå **DON'T**:
- Use cold, formal language
- Frame as vendor-customer
- Focus only on features
- Treat users as ticket numbers

**Examples**:

| ‚ùå Transactional | ‚úÖ Partnership |
|------------------|----------------|
| "Submit your information" | "Let's get you set up" |
| "Complete the following steps" | "Here's what we'll do together" |
| "Your subscription has been activated" | "Welcome! We're excited to work with you" |
| "Contact support for assistance" | "We're here if you need anything" |

## üé® Tone by Context

### Onboarding & Welcome

**Emotional State**: Uncertain, possibly overwhelmed  
**Tone**: Warm, reassuring, guiding

**Goals**:
- Reduce initial anxiety
- Build confidence quickly
- Set clear expectations
- Make first step obvious

**Copy Patterns**:
```
‚úÖ "You're in the right place. We'll guide you through everything step-by-step."
‚úÖ "Let's get you set up. This will take about 15 minutes."
‚úÖ "Welcome! We're here to make Michigan sick time compliance effortless."
‚úÖ "No worries if you're not sure about something ‚Äî we'll explain as we go."
```

### During Tasks (Guided Flows)

**Emotional State**: Focused, cautiously optimistic  
**Tone**: Patient, encouraging, clear

**Goals**:
- Maintain momentum
- Reduce cognitive load
- Celebrate progress
- Prevent abandonment

**Copy Patterns**:
```
‚úÖ "You're doing great! Just a few more questions."
‚úÖ "Step 3 of 5 ‚Äî more than halfway there!"
‚úÖ "Good news: as a small employer, you have simpler rules."
‚úÖ "üí° Why we ask this: [contextual explanation]"
```

### Success States

**Emotional State**: Relieved, proud  
**Tone**: Celebratory, validating, forward-looking

**Goals**:
- Acknowledge accomplishment
- Build confidence
- Encourage continued engagement
- Set up next steps

**Copy Patterns**:
```
‚úÖ "üéâ You're 100% compliant ‚Äî well done!"
‚úÖ "Setup complete! You can sleep easy tonight."
‚úÖ "Perfect! Your employees can now view their balances."
‚úÖ "You've successfully uploaded 8 employees. Great work!"
```

### Error States

**Emotional State**: Frustrated, possibly anxious  
**Tone**: Empathetic, solution-focused, calm

**Goals**:
- Acknowledge the problem
- Reduce frustration
- Provide clear fix
- Maintain trust

**Copy Patterns**:
```
‚úÖ "We noticed a small issue with your email. Let's fix that together."
‚úÖ "That password isn't quite strong enough. Try adding a number or symbol."
‚úÖ "Hmm, we couldn't connect. We'll retry automatically in a few seconds."
‚úÖ "This field needs a bit more information. Here's what we're looking for..."
```

### Security & Trust Moments

**Emotional State**: Cautious, evaluating  
**Tone**: Confident, transparent, protective

**Goals**:
- Build trust immediately
- Provide specific evidence
- Demonstrate expertise
- Reassure continuously

**Copy Patterns**:
```
‚úÖ "Your data is encrypted before it leaves your device."
‚úÖ "Bank-level security ‚Ä¢ AES-256 encryption ‚Ä¢ Google Cloud KMS"
‚úÖ "This calculation follows Michigan ESTA law Section 3(a) exactly."
‚úÖ "üîí All changes are logged for audit compliance."
```

### Complex or Technical Situations

**Emotional State**: Potentially overwhelmed  
**Tone**: Patient, simplifying, step-by-step

**Goals**:
- Break down complexity
- Prevent overwhelm
- Guide progressively
- Build understanding

**Copy Patterns**:
```
‚úÖ "This looks complicated, but we'll break it down step-by-step."
‚úÖ "In simple terms: [plain language explanation]"
‚úÖ "Here's what this means for you: [practical implication]"
‚úÖ "Don't worry about the legal details ‚Äî we've got that covered."
```

### Empty or First-Use States

**Emotional State**: Curious, exploratory  
**Tone**: Inviting, helpful, action-oriented

**Goals**:
- Guide first action
- Show value immediately
- Reduce empty-state anxiety
- Make next step obvious

**Copy Patterns**:
```
‚úÖ "Let's add your first employee to get started."
‚úÖ "Your dashboard will fill up once you complete setup."
‚úÖ "No sick time tracked yet. That's normal ‚Äî we'll start counting once you add hours."
‚úÖ "Ready to begin? Click here to start your guided setup."
```

## üìù Writing Guidelines

### Language Level

**Target**: 8th-grade reading level (Flesch-Kincaid)

**Tools**:
- [Hemingway Editor](http://www.hemingwayapp.com/)
- [Grammarly](https://www.grammarly.com/)
- [WebAIM Readability](https://wave.webaim.org/)

**Guidelines**:
- Short sentences (avg 15-20 words)
- Simple words over complex
- Active voice over passive
- Concrete over abstract

### Formatting for Clarity

‚úÖ **DO**:
- Use bullet points for lists
- Bold key information
- Break up long paragraphs
- Use headings for hierarchy

**Example**:
```
‚úÖ Good:
Your sick time policy:
‚Ä¢ Accrual: 1 hour per 30 worked
‚Ä¢ Maximum: 40 hours per year
‚Ä¢ Carryover: Up to 40 hours

‚ùå Bad:
Your policy includes accrual at 1 hour per 30 worked with a maximum of 40 hours per year and carryover up to 40 hours.
```

### Pronouns & Perspective

**Use "you" and "we"**:
- "You" = the user
- "We" = ESTA Tracker (the team, the system)
- "Let's" = collaborative action

**Examples**:
```
‚úÖ "You're all set up"
‚úÖ "We've calculated your accrual"
‚úÖ "Let's add your employees"
‚ùå "The user's setup is complete"
‚ùå "System has processed calculations"
```

### Call-to-Action (CTA) Language

**Make CTAs specific and action-oriented**:

| Context | ‚ùå Generic | ‚úÖ Specific |
|---------|-----------|-------------|
| Onboarding | "Next" | "Continue Setup ‚Üí" |
| Completion | "Done" | "View My Dashboard" |
| Adding Data | "Submit" | "Add Employee" |
| Help | "Learn More" | "Show Me How" |

### Microcopy & Labels

**Field labels should be questions or clear requests**:

```
‚úÖ "How many employees work for you?"
‚ùå "Employee Count"

‚úÖ "What's your business name?"
‚ùå "Business Name"

‚úÖ "When did you start operating in Michigan?"
‚ùå "Establishment Date"
```

## üö´ What to Avoid

### Fear-Based Language

‚ùå **Don't**:
- "Warning: Non-compliance penalties"
- "You must do this immediately"
- "Failure to complete will result in..."
- "Your account is at risk"

‚úÖ **Instead**:
- "Let's make sure you're compliant"
- "Complete this when you're ready"
- "Finishing this helps you stay protected"
- "Let's secure your account"

### Jargon & Acronyms

‚ùå **Don't**:
- "FTE", "ESTA", "MCL" (without explanation)
- "OAuth authentication"
- "Regulatory compliance parameters"
- "RBAC permissions"

‚úÖ **Instead**:
- "Full-time equivalent employees" (or "full-time employees")
- "Sign in"
- "Legal requirements"
- "Access permissions"

### Passive Voice

‚ùå **Don't**:
- "Your data has been encrypted"
- "The calculation was completed"
- "Approval is required"

‚úÖ **Instead**:
- "We encrypted your data"
- "We've calculated your accrual"
- "Your manager needs to approve this"

### Vague Promises

‚ùå **Don't**:
- "Best-in-class solution"
- "Fully optimized"
- "Guaranteed results"
- "Revolutionary platform"

‚úÖ **Instead**:
- "Automated sick time tracking"
- "Faster setup (15 minutes average)"
- "98% of users reach full compliance"
- "TurboTax-style guided setup"

## ‚úÖ Copy Review Checklist

Before finalizing any copy, check:

- [ ] **Tone**: Does it match the user's emotional state?
- [ ] **Clarity**: Can an 8th grader understand it?
- [ ] **Voice**: Does it sound like ESTA Tracker (reassuring, clear, empowering)?
- [ ] **Action**: Does it guide the user forward?
- [ ] **Trust**: Does it build confidence?
- [ ] **Simplicity**: Is it as short as possible while being clear?
- [ ] **Accessibility**: Is it screen-reader friendly?
